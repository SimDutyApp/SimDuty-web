Overview

This page explains how HOS timers work and how to stay compliant during active driving sessions.

Estimated time: 4 minutes Purpose: compliance-safe operation Output: lower HOS violation risk

Core concepts

  • Drive Left: remaining legal driving time before a required rest event.
  • Shift Left: remaining time in the current duty shift window.
  • Cycle Left: remaining time in your multi-day cycle limit.

Rest and break workflow

  1. Drive while monitoring remaining time against safety thresholds.
  2. Begin break or rest before the timer reaches a violation state.
  3. Resume driving only after HOS status returns to a compliant state.

Validation checkpoint

  • Drive Left and Shift Left are reviewed before route extension.
  • Break is started before timer reaches a violation state.
  • Status is compliant before resuming movement.

Quick support

  • If timers look frozen, verify SDK/NET connection first.
  • If status does not switch, check engine/vehicle state and movement threshold.
  • If limits seem inconsistent, confirm active policy and recent rest events.
